以下是按优先级排列的具体改进需求清单，每条均可独立实施或作为小项目里程碑：

### 📋 工程化与代码质量改进
1.**迁移至 TypeScript** - 配置 `tsconfig.json`，为现有 `*.js` 文件添加类型声明，解决类型错误。
2.**统一状态管理** - 用 Pinia 重构 `userStore.js`，与现有 `globalStore.js` 合并或明确职责，移除对 `localStorage` 的直接操作。
3.**搭建单元测试** - 引入 Vitest，为工具函数（如 `sqlExecutor.js`）和核心组件编写测试用例，目标覆盖率 > 80%。
4.**统一代码规范** - 配置 ESLint (eslint-config-standard-with-typescript) 和 Prettier，并集成 husky + lint-staged 实现 Git 提交前检查。
5.**实现错误边界** - 创建全局错误处理组件，捕获并优雅展示 SQL 执行等运行时错误。

### 🏗️ 架构与性能优化
6.**实现路由懒加载** - 修改路由配置，使用 `import()` 动态导入各页面组件，减少首屏资源体积。
7.**优化 Monaco Editor** - 配置 Vite 插件实现按需加载，仅引入 SQL 语言支持。
8.**实现关卡数据动态加载** - 将关卡 `*.js` 文件改为 JSON 或通过 API 获取，支持关卡热更新。
9.**封装可复用逻辑** - 创建 `useLevel`、`useAuth`、`useSqlExecutor` 等 Composables，替换现有 mixins 和重复代码。
10. **创建 API 抽象层** - 建立 `/src/api/` 目录，封装所有 HTTP 请求，为接入后端做准备。
11. **建立简易后端代理服务** - 使用 Express/Koa 创建 Node.js 服务，处理用户认证和未来 API 转发。

### 🤖 AI 与实时通信能力增强
12. **实现基础 AI 对话 (SSE)** - 创建后端 SSE 端点 (`/api/ai/stream/chat`) 和前端 `useSSE` Composable，实现与 OpenAI API 的流式对话。
13. **开发 AI 对话 UI 组件** - 创建 `AIChatBox.vue` 组件，集成消息展示、流式文本打字机效果和发送框。
14. **实现 SQL 智能解释功能** - 在后端 AI Service 中添加 Prompt 工程，实现“解释此 SQL”功能，并将结果流式返回前端。
15. **实现 SQL 智能优化建议功能** - 在后端 AI Service 中添加针对用户 SQL 语句的优化建议 Prompt，并返回格式化结果。
16. **升级为双向实时通信 (WebSocket)** - 将 SSE 连接升级为 WebSocket，实现后端主动推送、连接状态管理和双工通信。
17. **实现多轮对话上下文管理** - 在 `useAIChat` Composable 和后端服务中，维护并管理用户与 AI 的对话历史。
18. **实现多 AI 供应商支持** - 重构后端 AI Service，支持配置化切换 OpenAI、Claude、通义千问等不同供应商。
19. **实现 AI 使用统计与配额管理** - 在后端为用户添加 AI Token 消耗统计和简单的调用次数限制。

### ⚙️ 项目配置与部署
20. **完善环境变量管理** - 创建 `.env` 文件模板，区分开发、生产环境，安全管理 AI API Key 等敏感信息。
21. **优化构建与部署** - 配置 Dockerfile 和 CI/CD 流程，实现前后端的自动化构建与部署。
